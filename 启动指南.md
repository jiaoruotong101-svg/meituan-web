## 1. 项目是什么

这个项目分两部分：

```
美团酒店爬虫项目/
├── 📊 数据看板（网页）     ← 可视化查看爬到的酒店数据、统计图表、导出 CSV
│
├── 🕷️ 爬虫程序（Python）  ← 自动抓取美团酒店数据，写入数据库
│
└── 💾 数据库（SQLite）     ← 一个文件，存所有数据，两边共享
```

**简单说**：爬虫负责抓数据 → 存到数据库 → 看板读取数据库并展示。

## 2. 电脑需要装什么（首次，约 30 分钟）

⚠️ 这是**一次性**工作，装完以后就不用再装了。

### 2.1 安装 Node.js（看板运行环境）

1. 打开浏览器，访问 **https://nodejs.org/zh-cn**
2. 下载 **LTS 版本**（左边的绿色按钮，版本号 18.x 或 20.x 都行）
3. 双击安装包，**一路点「下一步」**，全部默认即可
4. 验证：打开「终端」（Mac）或「命令提示符」（Win），输入：
   ```
   node -v
   ```
   能看到版本号（如 `v20.11.0`）就成功了

### 2.2 安装 Bun（更快的运行工具，看板用这个启动）

1. 访问 **https://bun.sh**
2. 按页面提示安装：
   - **Mac**：终端里粘贴 `curl -fsSL https://bun.sh/install | bash` 回车
   - **Windows**：在 PowerShell 里粘贴 `irm bun.sh/install.ps1 | iex` 回车
3. 验证：终端输入 `bun -v`，能看到版本号即可

### 2.3 安装 Python（爬虫运行环境）

1. 访问 **https://www.python.org/downloads/**
2. 点黄色大按钮「Download Python 3.x.x」
3. 安装时 **务必勾选「Add Python to PATH」**（重要！）
4. 一路下一步
5. 验证：终端输入 `python --version`，能看到版本号即可

### 2.4 安装 ADB（连接手机用，爬虫需要）

1. **Mac**：终端输入 `brew install android-platform-tools`
   - 如果没装 brew，先访问 https://brew.sh 装 brew
2. **Windows**：
   - 下载 https://dl.google.com/android/repository/platform-tools-latest-windows.zip
   - 解压到比如 `C:\adb`
   - 把 `C:\adb` 加到系统环境变量 PATH（百度搜「Windows 添加环境变量」）
3. 验证：终端输入 `adb version`，能看到版本号即可

### 2.5 准备 Android 手机

⚠️ **必须是 Android 手机，iOS 不行**

1. 手机「设置」→「关于手机」→ 连续点「版本号」7 次，开启开发者模式
2. 「设置」→「系统」→「开发者选项」→ 打开「USB 调试」
3. 用 USB 线连电脑，手机会弹窗「是否允许 USB 调试」，选「允许」
4. 终端输入 `adb devices`，能看到一串设备号即可

## 3. 首次初始化（只做一次）

把项目文件夹放到一个好找的位置，比如：
- **Mac**：`/Users/你的用户名/美团酒店爬虫`
- **Windows**：`D:\美团酒店爬虫`

然后根据你的系统，**双击运行**对应的初始化脚本：

### Mac / Linux
打开终端，进入项目文件夹，运行：
```bash
bash setup.sh
```

### Windows
双击项目根目录的 `setup.bat` 文件

### 脚本会自动做什么
1. ✅ 安装看板前端依赖（`bun install`，约 2 分钟）
2. ✅ 创建数据库表结构（`bun run db:push`）
3. ✅ 生成 Prisma 客户端代码
4. ✅ 灌入 50 条示例数据（方便你先看效果）
5. ✅ 创建 Python 虚拟环境 + 安装爬虫依赖（约 3 分钟）

看到「✅ 初始化完成」就成功了。

## 5. 开始爬取数据（核心操作）

⚠️ 这是整个项目最复杂的部分，需要手机 + 证书 + 代理配合。

### 5.1 手机配代理（首次，约 20 分钟）

详细图文教程见：**`hotel_crawler/capture/cert_setup.md`**

简要步骤：
1. 电脑和手机连**同一个 WiFi**
2. 查电脑局域网 IP（Mac：`ifconfig`，Win：`ipconfig`），比如 `192.168.1.100`
3. 手机 WiFi 设置里，把 HTTP 代理改成「手动」：
   - 服务器：电脑 IP（如 `192.168.1.100`）
   - 端口：`8080`
4. 电脑启动 mitmproxy（见下一步），手机浏览器访问 `http://mitm.it` 下载证书
5. 把证书装到**系统级**（Android 7+ 必须，可能需要 root）

### 5.2 启动爬虫

#### 方式一：完整爬取（推荐，需要手机已配好代理和证书）

**Mac / Linux**：
```bash
bash start-crawler.sh
```

**Windows**：双击 `start-crawler.bat`

脚本会：
1. 启动 mitmproxy 监听 8080 端口（抓包）
2. 启动 uiautomator2 连接手机
3. 自动打开微信 → 美团小程序 → 酒店列表
4. 按城市依次爬取

#### 方式二：手动分步（更可控）

打开**两个终端**：

**终端 1 - 启动抓包**：
```bash
cd hotel_crawler
source .venv/bin/activate          # Mac
# Windows: .venv\Scripts\activate
mitmdump -s capture/mitm_addon.py --listen-port 8080
```

**终端 2 - 启动爬虫**：
```bash
cd hotel_crawler
source .venv/bin/activate          # Mac
# Windows: .venv\Scripts\activate
python main.py run --city 天津      # 爬天津
```

### 5.3 常用爬取命令

```bash
# 爬指定城市
python main.py run --city 天津
python main.py run --city 北京

# 按排期爬某天（Day1-Day7）
python main.py run --day 1

# 爬全部 15 个城市（会按排期分天）
python main.py run

# 只测试流程，不真的爬（验证环境用）
python main.py run --city 天津 --dry-run

# 看当前数据库统计
python main.py status

# 看断点续爬进度
python main.py checkpoint
```

### 5.4 爬取中的注意事项

1. **不要动手机**：爬取过程中手机会自动操作，手动触碰会打断
2. **保持屏幕常亮**：手机设置里关闭自动息屏
3. **别关电脑**：mitmproxy 和爬虫都要保持运行
4. **触发验证码**：看板「告警」标签会变红，立即停止，等 2 小时再继续
5. **爬取速度**：约 8-15 秒/家，200 家约需 40-50 分钟

### 5.5 字段映射调整（重要！）

⚠️ **首次使用前必须做这一步**，否则爬到的数据字段会错位。

`hotel_crawler/config/fields.yaml` 里的 API 字段映射是**占位符**，需要按真实抓包结果调整：

1. 启动 mitmproxy（见 5.2 方式二的终端 1）
2. 手机打开美团酒店小程序，进入酒店列表
3. mitmproxy 控制台会打印拦截到的 API URL 和 JSON
4. 对照 JSON 结构，修改 `fields.yaml` 里的字段路径
5. 修改后重启 mitmproxy 生效

**如果不会调整**：把抓到的 JSON 样本发给我，我帮你写映射。

## 7. 导出 CSV 给别人看

### 方式一：看板里导出（最简单）
1. 打开看板 http://localhost:3000
2. 点右上角「导出 CSV」按钮
3. 浏览器自动下载 CSV 文件

### 方式二：命令行导出
```bash
cd hotel_crawler
source .venv/bin/activate              # Mac
# Windows: .venv\Scripts\activate

python main.py export                  # 导出全部
python main.py export --city 天津      # 只导天津
```

导出的文件在：`hotel_crawler/output/hotel_YYYYMMDD.csv`

用 Excel 打开即可（编码 UTF-8 BOM，中文不乱码）。

## 8. 常见问题

### Q1: 双击 setup.bat 报错「bun 不是内部命令」
**A**: Bun 没装好或没重启电脑。重装 Bun（见 2.2），装完**重启电脑**让环境变量生效。

### Q2: 看板打不开，提示「无法访问此网站」
**A**:
1. 确认终端里 `start-dashboard` 脚本还在运行（别关终端窗口）
2. 等 30 秒再刷新
3. 手动访问 `http://localhost:3000`

### Q3: 爬虫报错「No devices found」
**A**: 手机没连上。
1. 确认 USB 线插好（有些线只能充电不能传数据，换一根试试）
2. 手机上确认「允许 USB 调试」
3. 终端输入 `adb devices` 确认能看到设备

### Q4: mitmproxy 抓不到包
**A**:
1. 手机和电脑是否在同一个 WiFi？
2. 手机代理是否设对了（电脑 IP + 端口 8080）？
3. 证书是否装到**系统级**（不是用户级）？
4. Android 7+ 必须系统级证书，详见 `hotel_crawler/capture/cert_setup.md`

### Q5: 爬虫跑起来但数据库没数据
**A**: 字段映射没配对。看 `hotel_crawler/logs/crawl_YYYYMMDD.log` 日志，找 mitmproxy 是否拦截到了请求。拦到了但没入库，说明 `fields.yaml` 字段路径不对。

### Q6: 触发验证码了怎么办
**A**:
1. **立即停止爬虫**（Ctrl+C）
2. 看板「告警」标签会显示告警
3. 等 2 小时后再继续
4. 如果反复触发，降低 `hotel_crawler/config/schedule.yaml` 里的 `per_hotel_interval_min` 和 `max`（改大间隔）

### Q7: 想中断后继续爬
**A**: 直接重新运行 `python main.py run --city 天津`，会自动从断点续爬（进度存在 `hotel_crawler/output/checkpoint.json`）。

### Q8: 想重新爬某个城市
**A**: 删掉 `hotel_crawler/output/checkpoint.json` 里对应城市的记录，再重新运行。

### Q9: 想清空所有数据重来
**A**:
```bash
# 删数据库文件
rm db/custom.db
# 重新建表
bun run db:push
# 重新灌示例数据（可选）
bun run scripts/seed-hotels.ts
```

### Q10: Mac 上提示「无法验证开发者」
**A**: 系统偏好设置 → 安全性与隐私 → 允许运行。

## 9. 关闭程序

### 关闭看板
在运行 `start-dashboard` 的终端里按 `Ctrl + C`

### 关闭爬虫
在运行爬虫的终端里按 `Ctrl + C`

### 关闭 mitmproxy
在运行 mitmproxy 的终端里按 `Ctrl + C`

### 取消手机代理
爬完后记得把手机 WiFi 代理改回「无」，否则手机上不了网。
